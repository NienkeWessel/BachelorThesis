\abstract{In order to prove properties of programming languages, we need to have a mathematical representation of the language. This thesis formalizes a subset of the Rust Programming Language. The focus is on ownership and borrowing, properties that are paramount to the memory safety that Rust claims to have. Besides our semantic rules, we have a compile time check system to model that Rust checks programs compile time. We explain our semantic rules and prove some properties of these rules, such as progress and preservation in combination with the compile time checker.}
